Imagine a life as an Data scientist where all Data is structured, readable and you got everything to your satisfaction. Wouldn't that be nice? One day I was tired of manually clustering my pdf documents into the right Label. It was a POC, where I wanted to analyze special document structures. Therefore I could serve a Model, which tells us which kind of structure Document is it.
The layout, in my imagination Documents do have different structured layouts, but for my Documents there was only 17 different structures. When you are able to classify which structured document you retrieve from a costumer, you would be able to classify more metadata out of this document. Costumer behaviour is important in the free market econmy.

So I came up with a simple Design where you can tell the system, how many Documenttype you got. You are able to name these and create costum folders in the background. On the left you can see the labels, with a small preview of the document, which helps you to store the document into the new label-folder. After you submitted the page with the correct labeld button, you get the next Documents on the right side. There is time to safe when you want to label documents and store it in the right folder.

Data preprocessing is the one of the exciting but frustrating part in ML projects, therefore I created this tool.

I did a description in a few sentence about this project for you:
Labeln is a performance tool for data cleaning in supervised machine learning projects. It visualize Documents and generates folder that maps an input to an labeled data category-folder. Dataset clearing should normally be Human-labeled data and for this approach Labeln is a good partner for the time consuming part of a ml-project.
