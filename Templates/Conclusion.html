"I do all the work, and they get all the praise." Ever heard this sentence from your colleague?
Take a step back and analyze the comment. There is a value in what your colleague is saying - you just have to listen.

One evening I dealt with this sentence. There is a lot of miscommunication in the sentence. At the end of the project, those involved can create a short recap so that everyone can understand the communication. Otherwise, the responsible person for the project sees himself and cannot keep up with other tasks. These processes are always common. If you know this introductory sentence, you can certainly imagine a similar situation.
This project was an azure project, where I wanted to handle requests/responses via the graph API from Microsoft.

In a few technical sentences: This is Web-App is for Users with the outh2, to grab Outlook attendees, Title and other Informations from a Meeting. The Writer is able to send the whole conclusion to all or only choosen attendees. It is a private take by hand project for my company, sometimes it is necessary to take an app to get on the right road. The purpose for this is app is quite simple: If you are talking for hours or even series dates, it is quite hard for all attendees to make a conclusion. It is a guide to make it more easy to remind people for the purpose conclusion. This will not work if you will clone it, you will first have to define the Secret of this App in Azure.
